🎯 Strategy Pattern em Ação

Este projeto demonstra o Strategy Pattern, um padrão de design que permite você escolher um algoritmo em tempo de execução.

📋 Interface (Contrato)

@FunctionalInterface
public interface MetodoPagamento {
    void processar(double valor) throws PagamentoException;
}

💳 Implementação 1: CartaoCredito

public class CartaoCredito implements MetodoPagamento {
    @Override
    public void processar(double valor) {
        // Validações de cartão
        // Autenticação
        // Processamento
    }
}

📱 Implementação 2: PIX

public class PIX implements MetodoPagamento {
    @Override
    public void processar(double valor) {
        // Validação de chave PIX
        // Processamento instantâneo
        // Sem limite
    }
}

🎯 Uso: Polimorfismo!

// Pedido não precisa saber qual tipo!
public void fecharPedido(MetodoPagamento metodo, double valor) {
    metodo.processar(valor);  // ← POLIMORFISMO!
}

// Funciona com qualquer implementação:
pedido.fecharPedido(new CartaoCredito("1234"), 100.0);
pedido.fecharPedido(new PIX("chave@pix"), 100.0);
pedido.fecharPedido(new PagamentoPix("chave-qr"), 100.0);
