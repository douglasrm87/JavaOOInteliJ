💡 DESTAQUES DO PROJETO

🎭 POLIMORFISMO EM AÇÃO
   → Quando pagamento é processado, método.processar() executa a versão correta
   → Mesmo interface, comportamentos diferentes (CartaoCredito vs PIX vs PagamentoPix)
   → Conceito mais importante de POO demonstrado na prática!

🏗️ ARQUITETURA MVC CLARA
   → Controller: DukeRevisaoController (7 rotas)
   → Model: Classes em faculdade/revisao/ (Produto, Pedido, etc)
   → View: Templates Thymeleaf em templates/revisao/

📊 DESIGN PATTERNS REAIS
   → Strategy: MetodoPagamento interface + 3 implementações
   → Template Method: Produto abstrata + Camisa concreta
   → State: StatusPedido enum com 5 estados
   → Composer: Objetos contêm outros objetos
   → MVC: Separação clara de responsabilidades

🎓 EDUCAÇÃO PRÁTICA
